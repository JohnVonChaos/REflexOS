┌──────────────────────────────────────────────────────────────────────────────┐
│                    DATA FLOW: INGEST → QUERY → GENERATE                      │
└──────────────────────────────────────────────────────────────────────────────┘

INGEST:
  text → tokenize → append to corpus
                 → update position index
                 → extract relations
                 → build syntactic edges
                 → calculate interference strengths

QUERY:
  query → expand via synsets
       → lookup positions
       → find interference
       → traverse graph
       → build entity profiles
       → score paths

GENERATE:
  interference hit → find sentence boundary
                  → generate from position
                  → filter via relational paths
                  → assemble output

Operation           Time          Space         Notes
─────────────────────────────────────────────────────────────────────────────

Interference        O(w₁×w₂×...)  O(1)         Cross-product of position sets

Generate            O(length)     O(tokens)     Linear scan from position

Total Query:        0.1-10ms      <1MB         Depends on corpus size + depth

Scalability:
• Tested: 100K tokens → 0.5-2ms queries
• Expected: 1M tokens → 5-20ms queries
• Bottleneck: Interference (cross-product)
• Solution: Position indexing + window limits
